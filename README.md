# PROJECT_RULES.md

> Menos código. Mais impacto.
>
> Boas ideias não precisam de mais código.
> Precisam de regras claras.

## O Problema

Ferramentas de IA são excelentes em gerar código.

Mas sem contexto e sem regras elas também são excelentes em:

* Criar abstrações desnecessárias
* Adicionar dependências sem necessidade
* Duplicar funcionalidades
* Construir soluções para problemas que ainda não existem
* Aumentar a complexidade do projeto a cada interação

O resultado?

Projetos mais difíceis de manter, revisar e evoluir.

---

## A Ideia

Inspirado pela filosofia do projeto Ponytail, este repositório propõe algo simples:

Antes de escrever código, tome decisões melhores.

O objetivo não é gerar menos código.

O objetivo é gerar apenas o código necessário.

---

## O Que é o PROJECT_RULES.md?

Um documento que fica na raiz do projeto e serve como referência para:

* IA
* Desenvolvedores
* Colaboradores
* Agentes autônomos
* Ferramentas de geração de código

Ele define princípios para reduzir complexidade e melhorar a qualidade das decisões durante o desenvolvimento.

---

## Como Usar

1. Baixe o arquivo `PROJECT_RULES.md`
2. Coloque na raiz do seu projeto

Exemplo:

```text
my-project/
│
├── PROJECT_RULES.md
├── README.md
├── src/
└── ...
```

3. Sempre forneça o arquivo como contexto para sua IA.

---

## Filosofia

Antes de criar algo novo pergunte:

1. Isso realmente precisa existir?
2. Já existe no projeto?
3. A linguagem já resolve?
4. O framework já resolve?
5. Uma dependência já resolve?
6. Dá para fazer em uma linha?
7. Só então criar algo novo.
